Overview

Implemented the Iterative Closest Point (ICP) algorithm for aligning 3D point clouds captured from different viewpoints. The algorithm iteratively finds point correspondences, computes optimal rigid transformations via SVD, and refines the alignment until convergence ‚Äî a fundamental operation in 3D reconstruction, LiDAR SLAM, and object pose estimation.

Problem

When a 3D sensor (LiDAR, depth camera) captures the same scene from two different poses, the resulting point clouds are in different coordinate frames. Aligning them requires estimating the rigid transformation (rotation + translation) that best maps one cloud onto the other. This is the point cloud registration problem, and it's central to building 3D maps from sequential scans.

Approach

The ICP algorithm alternates between two steps: (1) finding the closest point in the target cloud for each point in the source cloud, establishing correspondences, and (2) computing the rigid transformation that minimizes the sum of squared distances between corresponding point pairs using SVD-based closed-form solution.

The implementation includes KD-tree acceleration for nearest-neighbor queries, outlier rejection to handle partial overlap and noise, and convergence monitoring based on both transformation change and residual error. Multiple initial alignment strategies were tested to evaluate robustness to local minima.

Key Features

SVD-Based Alignment

Closed-form optimal rotation and translation estimation from matched point pairs.

KD-Tree Acceleration

Efficient nearest-neighbor search for scalable correspondence finding in large point clouds.

Outlier Rejection

Distance-based filtering to handle partial overlap and noisy correspondences.

Convergence Analysis

Monitoring of residual error and transformation delta across iterations to verify convergence.

Results

The ICP implementation successfully aligned point clouds from different viewpoints with sub-centimeter accuracy when given reasonable initial alignment. Convergence analysis showed exponential decrease in residual error over iterations, with typical convergence in 20-50 iterations. The KD-tree acceleration provided significant speedup over brute-force correspondence search for clouds with thousands of points.

What I Learned

Implementing ICP from scratch gave me a thorough understanding of rigid body alignment, SVD decomposition for transformation estimation, and the practical challenges of working with noisy 3D data. The project highlighted the importance of good initialization for iterative algorithms and the tradeoffs between correspondence rejection thresholds and convergence speed.
