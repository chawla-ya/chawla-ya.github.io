Overview

Developed a complete ROS2 driver package for a USB GPS receiver, including serial port communication, NMEA sentence parsing, custom ROS2 message definitions, and quantitative error analysis of position measurements. The driver publishes parsed GPS data as structured ROS2 topics for downstream consumption by navigation and localization nodes.

Problem

Integrating hardware sensors into the ROS2 ecosystem requires robust driver software that handles serial communication, protocol parsing, error checking, and data packaging into standard message formats. GPS receivers output raw NMEA sentences over serial, which must be parsed, validated, and converted into usable position data with proper coordinate frame conventions.

Approach

The driver node establishes a serial connection to the GPS receiver, reads incoming NMEA sentences (primarily GGA and RMC), validates checksums, and extracts position fields (latitude, longitude, altitude), timing, fix quality, and satellite count. Parsed data is packaged into custom ROS2 message types and published at the GPS update rate.

A quantitative error analysis was conducted by collecting stationary GPS data over an extended period, computing position statistics (mean, standard deviation, CEP), and characterizing error distributions in both open-sky and urban canyon environments to understand multipath and atmospheric effects.

Key Features

NMEA Parsing

Robust parser for GGA and RMC sentences with checksum validation and error handling.

Custom ROS2 Messages

Defined custom .msg types to package GPS data with all relevant fields for downstream nodes.

Launch File Configuration

Parameterized launch files for baud rate, serial port, and publishing frequency.

Error Analysis

Statistical characterization of GPS position accuracy including CEP, scatter plots, and time series.

Results

The driver reliably parsed and published GPS data at the receiver's native update rate with no dropped messages. Error analysis revealed characteristic patterns including multipath-induced position scatter in urban environments and the expected improvement in accuracy with increasing satellite count. The package was structured as a reusable ROS2 component with proper build system integration.
